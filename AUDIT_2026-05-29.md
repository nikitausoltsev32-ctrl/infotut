# Audit status — 2026-05-29

## Verified

- `npm run lint` passes.
- `npm run build` passes on Next.js 16.2.6.
- Public pages build: home, sections, article pages, feedback, privacy, terms, search, robots, sitemap.
- SEO basics are present: root metadata, `robots.txt`, `sitemap.xml`, public `/rss.xml` RSS endpoint.
- The obvious public 404 from `/about` links is fixed with a real `/about` page.
- Footer/header placeholder links were reduced to real routes.
- Search is visually aligned with section pages and queries `/api/articles` instead of duplicating article data in the client.
- Feedback API now validates input, rate-limits by IP, and sends mail through SMTP env when configured.
- Contact page includes a direct fallback email link plus editorial service links.

## Not ready for final business handoff

- Production env values still need to be set on hosting: `PAYLOAD_SECRET`, `DATABASE_URL`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHANNEL_ID`, `NEXT_PUBLIC_YANDEX_METRIKA_ID`, `SMTP_*`, `FEEDBACK_EMAIL`.
- Payload CMS collections exist, but live `/admin` and Payload REST routes have not been proven in this pass.
- Public pages, search, RSS, and sitemap still render mock article data, not CMS data.
- Telegram autoposting is code-ready from the Payload article publish hook, but it needs real bot/channel env and a live CMS publish test.
- Newsletter form is still a demo endpoint and does not persist subscribers.
- Some editorial/social claims and channel URLs are placeholder content and should be replaced with the client's real details before contractual delivery.
